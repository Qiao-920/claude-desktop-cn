# cc-haha 能力对照与 Claude Desktop CN 产品任务清单

参考仓库：[NanmiCoder/cc-haha](https://github.com/NanmiCoder/cc-haha)

这份文档只服务一个目的：把 `cc-haha` 里值得借鉴的能力拆成 Claude Desktop CN 自己的产品任务，而不是机械照搬。我们的定位仍然是中文桌面 Claude / Claude Code 客户端，优先把“本地工作区 + 命令执行 + 权限 + 预览”打磨成稳定闭环。

## 当前 P0 主线

### 1. 工作区

- 保留“选择工作区”作为 Code 页入口。
- 文件树支持浏览、打开、新建、删除、重命名。
- Git 仓库检测、分支、变更文件、单文件 diff 放在右侧控制台。
- 下一步补：最近工作区、工作区健康检查、项目一键打开 Code 工作区。

### 2. 命令执行

- 命令只在当前工作区目录执行。
- 安全模式禁用命令执行。
- 项目权限允许常规命令，但后端会阻止明显破坏性命令。
- 完全访问允许更宽操作，但仍需要在界面上明确风险。
- 本轮补：命令 quick actions、exit code、shell、权限模式、超时状态。

### 3. 权限

- 三档权限：
  - 安全模式：只读/编辑当前工作区文件，不执行命令。
  - 项目权限：当前工作区内文件操作 + 常规命令执行。
  - 完全访问：更宽文件与命令能力，适合可信本机工作流。
- 下一步补：权限变更日志、命令审计、危险命令二次确认持久化。

### 4. 预览稳定性

- HTML / React Artifact 预览必须使用沙箱 iframe。
- 预览要能识别三类状态：
  - 已正常加载。
  - 已加载但没有可见内容。
  - 脚本执行错误。
- 本轮补：预览加载状态、空白预警、脚本错误回传、手动刷新保留。

## 不直接照搬的部分

- 不把泄露源码路线作为宣传重点。
- 不优先做 CLI TUI，因为当前产品目标是桌面客户端。
- 不马上开放 Computer Use 全权限。这个能力很有价值，但必须配套权限、审计、确认、可撤销记录，否则对普通用户不够安全。
