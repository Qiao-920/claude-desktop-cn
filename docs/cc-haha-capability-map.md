# cc-haha 能力对照与 Claude Desktop CN 产品任务清单

参考仓库：[NanmiCoder/cc-haha](https://github.com/NanmiCoder/cc-haha)

当前主线仍然是：

- `P0 + P1` 一起推进

## 当前 P0 主线

### 1. 工作区

- 保留“选择工作区”作为 Code 页入口。
- 文件树支持浏览、打开、新建、删除、重命名。
- Git 仓库检测、分支、变更文件、单文件 diff 放在右侧控制台。
- 已补：最近工作区、工作区健康检查、项目一键打开 Code 工作区。
- 本轮已补：依赖安装状态、锁文件一致性、常用脚本覆盖、`.env` 模板检查和修复建议。

### 2. 命令执行

- 命令只在当前工作区目录执行。
- 安全模式禁用命令执行。
- 项目权限允许常规命令，但后端仍阻止明显破坏性命令。
- 完全访问允许更宽操作，但界面上保留风险提示。
- 已补：命令审计、风险命令审批、命令筛选与导出。

### 3. 权限

- 三档权限：安全模式 / 项目权限 / 完全访问。
- 权限模式会影响命令执行和部分文件操作。
- 高风险命令会要求确认并进入审计记录。

### 4. 预览稳定性

- HTML / React Artifact 使用沙箱 iframe。
- 预览可区分空白、脚本错误、console 日志、资源失败、请求失败。
- 当前重点从“能不能预览”，转到“能不能快速定位和恢复”。

## 当前 P1 主线

### 1. MCP 真功能化

- 设置页已是“真控制台”形态，不再只是配置壳。
- 已支持 stdio 工具发现、真实调用、参数表单、JSON 调用、调用审计。
- 下一步是 HTTP MCP 和更完整的调用详情。

### 2. Skills 中文化 / 防穿模

- 已有中文说明缓存、详情面板、英文原文回显。
- 已补项目绑定、触发示例、聊天菜单优先推荐。
- 已处理输入错位、路径穿模和内置来源脱敏。

### 3. Project / Cowork

- 已有项目页、项目文档、项目聊天、项目时间线。
- 已补项目状态：负责人、里程碑、下一步动作。
- 已补项目任务板：待处理 / 进行中 / 阻塞 / 已完成。
- Cowork 首页已具备项目摘要、时间线和跳转价值。

## 不直接照搬的部分

- 不把 CLI / TUI 作为主产品形态。
- 不把 Computer Use 全权限直接开放给普通用户。
- 不把“像源码泄露一样的路径暴露”当成卖点；桌面客户端仍以中文体验、稳定闭环和可维护性为主。
