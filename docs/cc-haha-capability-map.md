### 1. 工作区

- 保留“选择工作区”作为 Code 页入口。
- 文件树支持浏览、打开、新建、删除、重命名。
- Git 仓库检测、分支、变更文件、单文件 diff 放在右侧控制台。
- 已补：最近工作区、工作区健康检查、项目一键打开 Code 工作区。
- 本轮补：依赖安装状态、锁文件一致性、常用脚本覆盖、`.env` 模板检查和修复建议。

### 2. 命令执行

- 命令只在当前工作区目录执行。
- 安全模式禁用命令执行。
- 项目权限允许常规命令，但后端会阻止明显破坏性命令。
- 完全访问允许更宽操作，但仍需在界面上明确风险。
- 已补：quick actions、exit code、shell、权限模式、超时状态、中高风险审批和命令审计。
- 本轮补：审计筛选逻辑兼容已执行 / completed 历史记录。

### 3. 权限

- 三档权限：
  安全模式：只读 / 编辑当前工作区文件，不执行命令。
  项目权限：当前工作区内文件操作 + 常规命令执行。
  完全访问：更宽的文件与命令能力，适合可信本机工作流。
- 已补：命令审计、危险命令二次确认、按工作区过滤命令审计。
- 下一步补：权限变更日志、审计导出视图强化。

### 4. 预览稳定性

- HTML / React Artifact 预览必须使用沙箱 iframe。
- 预览要能识别四类信号：
  正常加载。
  已加载但没有可见内容。
  脚本执行错误。
  资源 / 请求加载失败。
- 已补：加载状态、空白预警、脚本错误回传、控制台日志、手动刷新保留。
- 本轮补：资源加载失败提示、fetch 失败提示、错误等级区分。

## 不直接照搬的部分

- 不把 CLI / TUI 作为主产品形态。
- 不把 Computer Use 全权限直接开放给普通用户。
- 不把“看起来像源码泄露的路线”当作产品卖点；桌面客户端还是以中文体验、稳定闭环和可维护性为主。
