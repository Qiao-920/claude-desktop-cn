# Claude Desktop CN v1.6.15

`1.6.15` 这一版是面向 Windows 稳定性的修复版，重点处理两个实际使用里最烦人的问题：后台托盘空白图标，以及偶发整窗黑屏。

## 本版重点

### 1. 修复 Windows 托盘空白图标

之前安装版进入后台后，托盘位置虽然存在，但图标会显示成一块空白。

这次改成了更稳的运行时资源路径：

- 不再直接读取开发目录里的 `public/`
- 托盘图标改为读取打包后稳定存在的资源
- 打包配置里同步补齐 Windows 运行时图标资源

### 2. 修复一个会引发黑屏的前端错误

排查日志时发现，消息区渲染里存在一个 `uiLanguage is not defined` 的作用域错误。

这个问题在特定渲染路径下会直接抛出 `ReferenceError`，从而造成页面渲染失败，表现出来就可能是：

- 页面大面积空白
- 窗口只有黑色背景
- 重新打开后又恢复

这一版已经把该错误改为标准 props 传递，不再引用越界变量。

### 3. 补上 Windows 稳定性兜底

为了优先把“黑屏壳子”这类问题压住，这版还补了几层保护：

- Windows 下关闭硬件加速
- 记录主进程 `did-fail-load`
- 记录 `render-process-gone`
- 在渲染进程异常退出时尝试自动恢复

## 安装包

```text
Claude-Desktop-CN-Setup-1.6.15.exe
```

## 备注

如果升级到 `1.6.15` 后托盘仍然出现异常，下一步就继续检查 Windows 托盘缓存和单独的多尺寸 `.ico` 内容，而不是再停留在路径层面。
