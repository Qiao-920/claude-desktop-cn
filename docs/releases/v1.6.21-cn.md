# Claude Desktop CN v1.6.21

这一版主要把 `cc-haha` 能力对照里的 MCP / Skills 主线继续往前推，并修复 Code 命令审计读取的兼容问题。它不是一次大范围重构，而是把前面已经搭好的设置页和能力清单继续变成可操作、可验证的产品功能。

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.21.exe`
- Release 页面：https://github.com/Qiao-920/claude-desktop-cn/releases/tag/v1.6.21

## 主要更新

### 1. MCP 工具发现

MCP 设置页不再只停留在服务器启停和连接测试：

- stdio MCP 服务支持“发现工具”。
- 会读取并保存工具列表、工具数量、最近扫描状态和扫描时间。
- 设置页会展示前几个工具名称，鼠标悬停可查看工具说明。
- 新增工具发现审计记录，方便回看最近一次发现结果。

HTTP MCP 的工具发现暂时标记为未接入，后续会单独补 HTTP 调用链路。

### 2. Skills 说明语言同步

Skills 设置页里的“说明语言”现在会同步到聊天输入区的 Skills 菜单：

- 设置为中文时，聊天菜单和悬停说明优先显示中文解释。
- 设置为英文时，保留原始英文说明。
- 设置变化会即时通知聊天页，不需要重启客户端。

### 3. Code 命令审计兼容修复

修复命令审计接口字段不一致的问题：

- 后端同时返回 `entries` 和旧字段 `audit`。
- 前端读取时兼容两个字段。
- Code 页右侧命令审计列表会更稳定地显示最近命令记录。

### 4. 能力对照表更新

`docs/cc-haha-capability-map.md` 已同步更新当前完成度：

- MCP 服务器：55%
- Skills 能力：60%
- Code 工作区闭环：62%

整体复现路线继续优先围绕“工作区 / 命令执行 / 权限 / 预览稳定性”推进。
