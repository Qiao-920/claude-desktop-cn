# Claude Desktop CN v1.6.20

这一版继续沿着“工作区 / 命令执行 / 权限 / 预览稳定性”主线推进，同时把 MCP、Skills 和 Cowork 从骨架页补成可操作的第一版产品功能。

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.20.exe`
- Release 页面：https://github.com/Qiao-920/claude-desktop-cn/releases/tag/v1.6.20

## 主要更新

### 1. Code 工作区健康检查

Code 页现在会根据当前工作区自动检测：

- 是否存在目录。
- 是否是 Git 仓库。
- 是否包含 `package.json`。
- 包管理器：npm / pnpm / yarn / bun。
- 可运行脚本和建议命令。
- 常见风险提示。

健康检查结果会显示在右侧工作区控制台里，常用命令快捷入口也会优先使用检测到的脚本。

### 2. 命令执行审批与审计

命令执行接口继续收紧：

- 安全模式仍然禁止执行命令。
- 高风险命令会被拦截。
- 中高风险命令需要在客户端确认后才会执行。
- 每次执行、失败、拦截、待确认都会写入审计记录。
- Code 页右侧会展示最近命令审计和风险等级。

这一步让“完全访问”不再只是一个开关，而是带有确认和记录的工作流。

### 3. 预览系统继续加固

Artifact / HTML 预览现在除了加载、空白和脚本错误状态，还会捕获预览 iframe 内的 `console.log` / `console.warn` / `console.error`。

遇到白屏、空白、模型生成错误脚本时，预览面板会显示日志，方便判断是客户端问题、脚本问题，还是生成内容本身有问题。

### 4. MCP 设置页变成真实功能

设置页里的 MCP 服务器不再只是说明：

- 支持新增 stdio / HTTP MCP 服务。
- 支持启用 / 禁用。
- 支持删除。
- 支持基础连通性测试。
- 支持查看最后一次测试状态。

后续会继续补环境变量、工具列表、服务日志和调用审计。

### 5. Skills 设置页变成真实功能

新增 Skills 设置页：

- 展示内置 Skills 和自定义 Skills。
- 显示启用数量、内置数量、自定义数量。
- 支持在设置页直接启用 / 禁用 Skill。

后续会补中文 tooltip、详情页、触发示例和项目绑定。

### 6. Cowork 任务看板

Cowork 页新增本地任务看板：

- 待处理 / 进行中 / 已完成三列。
- 支持添加任务。
- 支持在列之间移动任务。
- 支持重置模板。
- 任务会保存在本地。

这让 Cowork 从“说明页”开始变成真实的项目协作入口。
