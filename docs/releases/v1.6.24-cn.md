# Claude Desktop CN v1.6.24

这一版开始把 `Project / Cowork` 往你想要的“Codex 式项目驱动工作流”推进，不再只停留在项目列表和占位看板，而是补上了真正能用的项目工作区入口。

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.24.exe`

## 主要更新

### 1. Projects 开始具备“项目驱动入口”

项目列表和项目详情现在都补上了围绕工作区的关键动作：

- 绑定当前 `Code` 工作区到项目
- 直接为项目选择本地目录
- 在资源管理器中打开项目目录
- 复制项目工作目录
- 复制项目 deeplink
- 为项目新建聊天
- 直接“派生到本地 Code”
- 直接“派生到新工作树”

这意味着 `Projects` 不再只是存资料的地方，而开始成为真正的“项目中枢”。

### 2. 新工作树派生已接到后端

这版新增了项目工作区派生接口：

- 如果项目目录本身是 Git 仓库，会优先创建真正的 `git worktree`
- 如果当前目录不是 Git 仓库，会自动回退成一个新的本地工作区副本

这样既保留了你喜欢的 worktree 工作流，也避免在普通目录上点完直接报错。

### 3. Cowork 可以直接跳进具体项目

`Cowork` 页里的最近项目入口，不再只是跳回笼统的 Projects 页，而是可以直接打开指定项目。  
这一步虽然还不是完整的“协作中心”，但已经把 `Cowork -> Project` 这条路打通了。

### 4. 项目聊天输入继续补稳

`Projects` 详情页里的聊天输入框，之前也沿用了技能高亮叠层方案。  
这版一起改回单层 textarea 渲染，避免项目聊天里再次出现 slash skill 引用时的光标和输入错位。

## 当前判断

这一版之后，`Project / Cowork` 已经从“页面壳子”推进到了“项目工作流第一版”：

- `Projects`：开始像一个真正的项目容器
- `Cowork`：开始像一个项目总览和路由层
- `Code`：现在可以通过项目直接派生和切换

下一步最适合继续补的是：

- 项目文档再增强成长期说明卡
- 项目下聊天分组与更多快捷动作
- `Cowork` 的项目时间线 / 审阅状态 / 任务联动
