# Claude Desktop CN v1.6.26

这一版把 `Project / Cowork` 继续往真正可推进工作的项目操作台推进了一整块，同时把上一轮遗留的项目流页面英文和历史乱码做了一轮集中清理。

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.26.exe`

## 主要更新

### 1. Project 开始具备“推进项目”的状态层

`Projects` 不再只是项目列表和说明页，这一版补上了更接近真实工作流的推进字段：

- 项目状态：`进行中 / 阻塞 / 待发布 / 已完成`
- 负责人
- 当前里程碑
- 下一步动作

这些字段已经进入前后端数据结构，项目详情里可以直接编辑和保存，后续也能自然接到时间线和协作视图。

### 2. 项目任务板第一版

项目详情页现在已经有本地任务板：

- 待处理
- 进行中
- 阻塞
- 已完成

支持新增任务、移动状态、删除任务。这样项目页不再只是“资料容器”，而开始具备承接实际工作推进的能力。

### 3. Cowork 首页摘要卡继续增强

`Cowork` 首页的项目摘要卡已经开始显示真正有决策价值的信息：

- 项目概览
- 目标 / 约束 / 常用命令
- 工作区、聊天数、文件数、GitHub 来源数
- 项目状态
- 负责人
- 里程碑
- 下一步动作
- 任务分布

这让 `Cowork` 更像项目总览和路由层，而不是单纯占位页。

### 4. 项目时间线和项目聊天继续闭环

项目详情页里这一版继续强化了项目视角：

- 项目时间线统一收拢文档、聊天、文件、GitHub 同步事件
- 项目聊天支持更多项目内操作
- 派生到本地 Code / 派生到新工作树继续保留在项目动作里

这条线会继续往你喜欢的 `Codex` 式项目驱动工作流靠拢。

### 5. 页面汉化和历史乱码集中清理

这版顺手清掉了一批还会直接暴露在界面上的英文和历史乱码，重点覆盖：

- 项目页
- Cowork 页
- 聊天页
- 自定义页
- 作品页

包括项目搜索、排序、按钮、菜单、项目文档、GitHub 来源、聊天操作、Cowork 摘要等一批直接影响观感的文字。

## 当前判断

`v1.6.26` 之后，`Project / Cowork` 已经不只是“展示入口”，而是开始进入真正的项目操作台阶段：

- `Projects`：能承接项目说明、项目状态、任务和聊天
- `Cowork`：能承接总览、摘要、时间线和跳转
- `Code`：继续作为本地工作区和命令执行闭环

下一步比较顺的方向就是把这套项目层继续往下接：

- 项目任务和聊天 / 时间线的联动
- GitHub 最小接入深化
- 项目记忆继续并入项目文档
