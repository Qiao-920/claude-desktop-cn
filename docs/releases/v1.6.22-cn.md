# Claude Desktop CN v1.6.22

这一版聚焦 `P0` 主线收口，同时保留前两轮已经推进的 `MCP 真功能化` 和 `Skills 中文化 / 防穿模` 成果。重点不是堆页面，而是把“工作区 + 命令执行 + 权限 + 预览”继续往稳定闭环推进。

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.22.exe`
- Release 页面：[v1.6.22](https://github.com/Qiao-920/claude-desktop-cn/releases/tag/v1.6.22)

## 主要更新

### 1. Code 工作区健康检查升级

Code 页右侧的工作区健康检查不再只是识别 `package.json` 和脚本列表，而是补成了更接近真实项目诊断的版本：

- 检查 `node_modules` 是否缺失
- 检查锁文件是否缺失或冲突
- 检查 `package.json.packageManager` 和锁文件是否一致
- 检查常用脚本覆盖情况，例如 `build` / `dev` / `test` / `typecheck`
- 检查 `.env.example` 等模板是否存在但本地环境文件缺失
- 把这些结果整理成“修复建议”，并可一键填入命令框

这让 Code 页从“看状态”变成了“能指出问题和下一步怎么修”。

### 2. 预览稳定性继续硬化

Artifact 预览这轮补上了更细的错误感知：

- 区分脚本执行错误和资源加载失败
- 识别 `fetch` 请求失败
- 在预览层给出错误等级提示
- 继续保留 console 日志和空白预览检测

现在遇到外部 CDN、脚本依赖或接口请求出错时，不再只剩一句笼统的“预览失败”。

### 3. MCP 真功能化正式进入可用阶段

前两轮已完成并并入本版：

- stdio MCP 服务支持真实 `tools/call`
- 设置页支持工具列表、参数表单和 JSON 调用
- 返回调用结果摘要
- 保留调用审计记录

这意味着 MCP 不再只是“能发现工具”，而是已经进入“能真实调一次”的阶段。

### 4. Skills 中文化 / 防穿模继续完善

前两轮已完成并并入本版：

- Skills 中文说明由后端缓存管理
- 设置页展示中文说明、英文原文和翻译状态
- Skills 支持项目绑定和触发示例
- 聊天菜单会优先展示当前项目绑定的 Skill
- 若输入框为空，点击 Skill 可直接带出触发示例

这一轮让 Skills 从“设置页能看见”推进到“聊天里真能用、中文链路不穿模”。

### 5. 命令审计细节修正

- 命令审计筛选兼容历史 `completed` 和当前 `executed`
- 风险筛选兼容正常命令的低风险展示
- 审计导出继续保留
