# Claude Desktop CN v1.6.29

这一版继续把产品体验往 `Codex` 的工作台形态推进，不再只是补单点功能，而是把项目结构、自动化和能力入口一起收紧成更可日用的桌面工作流。

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.29.exe`

## 主要更新

### 1. 项目创建改成“先选目录”

`新建项目` 现在优先像 Codex 一样先打开本地文件夹选择器，再把你选中的目录绑定为项目工作区；不再默认“凭空建一个项目”。

- 选完文件夹后会默认用文件夹名作为项目名
- 仍然保留“创建空白项目”作为备用入口
- 项目菜单里可以重新选择目录、在资源管理器中打开、复制项目路径

### 2. 项目树 / 项目聊天更像 Codex

左侧项目树和项目聊天菜单这轮专门做了紧凑化：

- 项目树支持像 Codex 一样展开 / 收起项目聊天
- 当前激活项目也能正常折叠，不再点了没反应
- 项目行和聊天行更紧凑，聊天类型徽标改成单行内联显示
- 项目菜单和项目聊天菜单继续保留“打开项目 / 新建聊天 / 目录操作”等入口

### 3. 多 Agent 工作台补成可见入口

这轮不再只留后端字段，前端已经能直接看见和使用：

- 左侧新增 `多 Agent`
- 支持项目级 Agent 卡片总览
- 支持内置 Agent 模板
- 支持从工作台直接给项目新增 Agent、启动会话、打开最近会话、执行任务

### 4. 自动化页压紧并往 Codex 靠

自动化页不再是“松散大卡片 + 大块空白”的样子，而是更像任务工作台：

- 顶部说明区和统计卡压缩
- 模板区改成更密集的任务面板
- 最近活动区改成真正的侧栏信息区
- 跨项目过滤和模板使用链路继续保留

### 5. 自定义页推进成“插件 / 能力中心”

这轮新增了更像 Codex 的统一能力入口：

- 在 `自定义` 里加入 `插件` 视图
- 把 GitHub、Computer Use、Skills、多 Agent、自动化、连接器做成统一入口卡片
- 可以从这里继续跳转到具体的 Skills / 连接器 / 多 Agent / 自动化工作流

这还是第一版“能力中心”，不是完整插件商城；下一步可以继续往“已安装 / 推荐 / 搜索 / 详情页”推进。

### 6. 稳定性修复

- 修复 `Projects` 页因图标未导入导致的页面加载失败
- 修复 `多 Agent` 页被项目页异常连带拖崩的问题
- 修复项目树当前项目无法折叠的问题

## 当前判断

`v1.6.29` 之后，这个分支已经更明显地从“中文化 Claude 桌面壳”转向“偏 Codex 的中文桌面工作台”：

- 项目不再只是资料卡，而是开始围绕真实目录组织
- 项目树和项目聊天开始形成 `项目 -> 多会话` 结构
- 多 Agent、自动化、自定义能力入口都已经能看见、能点、能继续往下做
