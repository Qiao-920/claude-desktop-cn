# Claude Desktop CN v1.6.25

这一版把 `v1.6.24` 打下的 `Project / Cowork` 基础继续往前推了一整块，重点不再只是“项目入口”，而是开始形成更像 Codex 的桌面工作流：

- 桌面多标签工作流
- Project doc 结构化项目文档
- 项目聊天分组与时间线
- Cowork 首页的项目文档摘要卡

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.25.exe`

## 主要更新

### 1. 桌面端开始支持多标签工作流

这版把桌面工作区入口从单一页面，推进成了真正的桌面多标签流：

- `聊天 / 协作 / 代码` 可以作为独立标签存在
- 标签支持独立标题、固定、关闭和重排
- 路由会记住当前 tab，上下文切换更稳定
- `Code` 和聊天草稿开始按桌面 tab 隔离保存，减少相互覆盖

这一步的意义是把“桌面客户端”真正从单页切换，推进到更接近本地 IDE / Codex 的工作节奏。

### 2. Project doc 升级成结构化项目文档

`Projects` 里的项目说明不再只是一个大文本框，而是变成了结构化的项目文档编辑器：

- 项目概览
- 项目目标
- 技术栈 / 运行方式
- 约束 / 原则
- 常用命令
- 补充备注

保存后会自动回写成项目 instructions，同时在项目详情页顶部拆块展示。这样项目文档开始真正承担“长期上下文容器”的角色，而不是只做附注。

### 3. 项目聊天分组和项目时间线已经接起来

项目详情页里，这版继续把聊天和活动做成更像工作流的样子：

- 项目聊天按时间分组
- 聊天支持打开、派生到本地 Code、复制会话 ID、复制 Deeplink、移出项目
- 原来的最近活动升级成 `项目时间线`
- 时间线会统一显示项目文档、聊天、文件、GitHub 同步等事件

这样单个项目页不再只是资料堆积，而是开始具备“可回顾、可跳转、可追踪”的推进感。

### 4. Cowork 首页开始具备项目文档摘要能力

`Cowork` 这版不只是总览页，还开始承担“项目路由层”的角色：

- 首页新增 `项目文档摘要` 卡片
- 直接显示项目概览、目标、约束、常用命令
- 同时补上聊天数、文件数、GitHub 来源数和工作区信息
- 点卡片即可跳到对应项目

除此之外，`Cowork` 里的项目最近活动也升级成了按时间分组的 `项目时间线`，和项目详情页形成统一体验。

## 当前判断

`v1.6.25` 之后，我们的桌面工作流已经明显更接近你要的方向：

- `App`：开始具备桌面多标签工作流
- `Projects`：开始像真正的项目中枢
- `Cowork`：开始像项目总览和路由层
- `Code`：开始按桌面标签保存工作区与命令上下文

换句话说，这一版的重点不是单点修补，而是把“桌面标签 + 项目文档 + 项目时间线 + Cowork 摘要卡”连成了一条更完整的本地工作流。
