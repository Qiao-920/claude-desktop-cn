# Claude Desktop CN v1.6.23

这一版继续收口 `P1`，重点不是铺新页面，而是把已经落地的 `Skills` 和 `Code` 体验补稳，修掉几个会直接影响日用的明显 bug。

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.23.exe`
- Release 页面：[v1.6.23](https://github.com/Qiao-920/claude-desktop-cn/releases/tag/v1.6.23)

## 主要更新

### 1. 修复 Skills 引用时输入框错位

聊天输入框里原先为了把 `/skill-name` 做成高亮，使用了覆盖层去镜像文本。这个方案在中文输入和连续打字时容易造成：

- 光标位置和显示文字错位
- 高亮层与真实输入层上下重叠
- 输入体验发飘，尤其是引用 Skill 之后继续打字时

这版已经把这套覆盖层去掉，改回稳定的真实文本输入，优先保证引用 Skill 之后的输入、光标和中文 IME 组合输入正常。

### 2. 修复 Skills 详情“穿模”

Settings 里的 Skills 详情页之前会把内置 Skill 的真实打包路径直接显示出来，例如 `app.asar` 下的本机绝对路径。这会把实现细节和本地安装结构直接暴露给用户，属于明显穿模。

现在已经改成：

- 内置 Skill 只显示脱敏后的来源标签，例如 `内置 Skills / code-review`
- 不再在详情提示里暴露打包后的本机绝对路径
- 内置 Skill 默认不再展示“打开目录”入口
- 用户自定义 Skill 仍然保留可读的本地来源信息

### 3. 修复 Code 页面黑屏

这轮把 Code 页相关回归一起收掉：

- 把工作区健康检查里的“修复建议”块放回正确位置
- 避免错误内容误插进编辑区空状态
- 给 `代码` 页增加错误边界兜底

即使后续还有局部渲染异常，也不会再直接把整页打成黑屏。
