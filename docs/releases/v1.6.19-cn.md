# Claude Desktop CN v1.6.19

这一版把产品路线继续往 Code 工作区主线推进：先对照 `cc-haha` 梳理能力地图，再把“工作区 / 命令执行 / 权限 / 预览稳定性”做成可用的第一轮闭环。

## 下载

- Windows 安装包：`Claude-Desktop-CN-Setup-1.6.19.exe`
- Release 页面：https://github.com/Qiao-920/claude-desktop-cn/releases/tag/v1.6.19

## 主要更新

### 1. cc-haha 能力对照表

新增 `docs/cc-haha-capability-map.md`，把 `cc-haha` 的 TUI、MCP、Skills、权限、Diff、多 Agent、桌面端和远程通道能力拆成 Claude Desktop CN 自己的任务清单。

这份清单不是照抄路线，而是按我们的产品定位拆成：

- P0：工作区、命令执行、权限、预览稳定性。
- P1：MCP、Skills、模型管理、项目记忆。
- P2：多 Agent、远程 Channel、Computer Use、定时任务。

### 2. Code 工作区命令入口

Code 工作区新增常用命令快捷入口，支持快速插入：

- `dir`
- `git status --short --branch`
- `npm ls --depth=0`
- `npm test`
- `npm run build`

这些入口先服务“快速检查项目状态”，后续会继续升级成工作区健康检查和脚本识别。

### 3. 后端权限守卫

命令执行接口开始真正校验权限模式：

- `安全模式` 下禁止直接执行命令。
- 非 `完全访问` 会拦截明显高风险命令。
- 命令超时会被限制在合理范围内。

这一步的目标是让权限按钮不只是 UI，而是真的影响后端行为。

### 4. 命令结果更清楚

命令执行结果现在会补充：

- Shell
- 退出码
- 耗时
- 超时状态
- 权限模式
- 工作目录

这样更容易判断问题来源：是命令失败、权限拦截、脚本无输出，还是模型生成了不合适的命令。

### 5. Artifact / HTML 预览稳定性

Artifact / HTML 预览改成沙盒 `srcDoc` 流程，并加入加载、空白和错误状态检测。

这能减少两类常见问题：

- 页面黑屏但不知道为什么。
- HTML 实际为空或脚本错误，却看起来像客户端没反应。
