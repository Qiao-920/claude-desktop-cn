# Claude Desktop CN v1.6.19

这一版把产品路线继续往 Code 工作区主线推进：先对照 `cc-haha` 梳理能力地图，再把“工作区 / 命令执行 / 权限 / 预览稳定性”做成可用的第一轮闭环。

## 主要更新

1. 新增 `docs/cc-haha-capability-map.md`，把 cc-haha 的 TUI、MCP、Skills、权限、Diff、多 Agent、桌面端和远程通道能力拆成 Claude Desktop CN 自己的任务清单。
2. Code 工作区新增常用命令快捷入口，支持快速插入 `dir`、`git status`、依赖检查、测试、构建等命令。
3. 命令执行接口开始真正校验权限模式：`项目权限` 禁止直接执行命令，非 `完全访问` 会拦截高风险命令。
4. 命令结果补充 Shell、退出码、耗时、超时、权限模式和工作目录等信息，排查失败原因更直接。
5. Artifact / HTML 预览改成沙盒 `srcDoc` 流程，并加入加载、空白和错误状态检测，降低黑屏和空白预览的概率。

## 验证

- `npm run build`
- `npm run electron:build:win`
